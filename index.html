Meet Pico

Pico is a state-of-the-art mobile robot designed to seamlessly integrate into your indoor environment. Equipped with a versatile robotic arm and omnidirectional wheels, Pico is engineered for precision, adaptability, and intelligent operation.

Key Features & Abilities

  • Advanced Robotic Arm: 6-axis robotic arm with high precision for intricate tasks, capable of lifting up to 2 kg.
  • AI-Powered Navigation: Cameras for accurate mapping, obstacle avoidance, and autonomous path planning.
  • Voice & Gesture Control: Intuitive control options for easy interaction and task assignment.
  • Modular Design: Expandable ports for additional sensors or tools, allowing for future upgrades and customization.
  • Long-lasting Battery: Up to 3 hours of continuous operation on a single charge, with quick-charge capabilities.
  • Object Recognition: Integrated computer vision for identifying and interacting with various objects.

Medicine Assistant Roadmap

Steps to enable Pico to recognize medicines and fetch them autonomously. Includes both design/hardware and coding phases.

Phase 1 – Design & Hardware Setup

  • Design chassis for stability and space for robotic arm.
  • Select motors and wheels for smooth indoor movement.
  • Assemble robotic arm, add limit switches or sensors for safe gripping.
  • Set a home docking location for charging and start/end positions.

Phase 2 – Coding & Vision

  • Implement OCR for reading medicine labels using Tesseract or EasyOCR + Coral TPU.
  • Create a medicine database with all known medicine names.
  • Use object detection (YOLOv5 / MobileNet) to locate medicine boxes/bottles.
  • Add lighting for consistent OCR accuracy.

Phase 3 – Autonomous Fetching & Navigation

  • Map key locations: medicine shelf, delivery point, table. (SLAM with ROS or OpenCV)
  • Enable safe navigation with obstacle detection and avoidance.
  • Implement task flow: Command → Search → Navigate → Grab → Deliver.
  • Handle missing medicine: alert user via dashboard or voice.

Phase 4 – Testing, Safety & Refinement

  • Test arm gripping reliability, add retries for failures.
  • Validate OCR results before grabbing medicine.
  • Add voice feedback: "Fetching..." and "Delivered". (pyttsx3 / gTTS)
  • Optional: Face recognition for personalized delivery (OpenCV + face_recognition).
  • Optimize battery usage, mobility, and arm speed for efficiency.
