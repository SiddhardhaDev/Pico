Meet Pico

Pico is a state-of-the-art mobile robot designed to seamlessly integrate into your indoor environment. Equipped with a versatile robotic arm and omnidirectional wheels, Pico is engineered for precision, adaptability, and intelligent operation.

Key Features & Abilities

  • Advanced Robotic Arm: 6-axis robotic arm with high precision for intricate tasks, capable of lifting up to 2 kg.
  • AI-Powered Navigation: Cameras for accurate mapping, obstacle avoidance, and autonomous path planning.
  • Voice & Gesture Control: Intuitive control options for easy interaction and task assignment.
  • Modular Design: Expandable ports for additional sensors or tools, allowing for future upgrades and customization.
  • Long-lasting Battery: Up to 3 hours of continuous operation on a single charge, with quick-charge capabilities.
  • Object Recognition: Integrated computer vision for identifying and interacting with various objects.

Pico Development Roadmap – Detailed Phases

Phase 1 – 3D Designing

In this phase, we model Pico in CAD software to ensure mechanical stability and proper component placement. Details include:

  • Wheels: Place two powered wheels at the back with sufficient diameter for smooth indoor mobility.
  • Castor Wheel: One front castor for balance and steering flexibility.
  • Battery: Central placement to lower center of gravity and avoid tipping during arm movement.
  • Controller Board (Raspberry Pi & Motor Drivers): Positioned near battery for easy cabling and heat dissipation.
  • Robotic Arm Mount: Fixed on top plate with support brackets; check arm reach and rotational clearance.
  • Sensor Mounts: Camera at front, IR/ultrasonic sensors on sides, distance from edges to avoid blind spots.
  • Arm Storage: Slot or cavity for arm when not in use to reduce robot footprint.
  • Ventilation & Cable Channels: Ensure airflow and tidy wiring in CAD model.
  • Simulation: Use CAD simulation tools to verify center of gravity, stability, and wheel torque requirements.

Phase 2 – Physical Assembly

Construct Pico according to the 3D design. Steps:

  • Print 3D chassis, arm brackets, and battery holders with strong PLA/ABS.
  • Install powered wheels and castor, verify alignment and rotation freedom.
  • Mount Raspberry Pi and motor drivers securely with screws/standoffs.
  • Place battery in central cavity; add Velcro or custom holder for stability.
  • Route wires: motor cables, sensors, cameras, power lines. Keep clear of moving parts.
  • Install robotic arm, connect motors/servos; test movement limits and repeatability.
  • Fix camera, IR, and ultrasonic sensors; calibrate range and angles.
  • Test rolling movement without load; adjust wheel positions if necessary.
  • Check arm reach and ability to grab objects on flat surfaces and shelves.
  • Final assembly: tighten all screws, secure loose wires, and check balance.

Phase 3 – Basic Coding & System Initialization

Software setup and initialization:

  • Install OS on Pi and set up Python environment with necessary libraries (OpenCV, PySerial, TensorFlow Lite).
  • Write initialization scripts for motors, sensors, camera, and robotic arm.
  • Implement logging system to track movements, sensor readings, and errors.
  • Flowchart logic: Start → Initialize hardware → Run self-checks → Ready for commands.
  • First scripts: simple motor forward/back, arm up/down, sensor readouts.
  • Debugging: Check motor directions, verify sensor ranges, camera feed validation.

Phase 4 – Navigation & Mapping

Enable Pico to navigate autonomously:

  • Set up SLAM (Simultaneous Localization and Mapping) using ROS/OpenCV.
  • Create a digital map of indoor space with coordinates for shelves, tables, and docking station.
  • Write path-planning algorithms to avoid obstacles dynamically.
  • Flowchart logic: Receive command → Map path → Check for obstacles → Move → Update map.
  • Debugging: Test navigation around furniture, verify accuracy of localization, adjust sensor offsets.

Phase 5 – Object Recognition & OCR

Implement recognition of objects and medicine labels:

  • Set up camera feed and preprocess images for lighting normalization.
  • Train or use pre-trained models (YOLOv5, MobileNet) to detect medicine boxes/bottles.
  • Implement OCR with Tesseract/EasyOCR to read medicine names from labels.
  • Flowchart: Capture image → Object detection → Crop detected object → OCR → Match with database → Confirm correct item.
  • Debugging: Test multiple medicine types, angles, lighting conditions, and label fonts.

Phase 6 – Task Execution & User Feedback

Final integration: Pico can receive commands and fetch objects:

  • Implement task execution flow: Receive request → Navigate → Identify object → Grab → Deliver → Return to dock.
  • Add voice feedback using pyttsx3/gTTS: “Fetching medicine”, “Delivered successfully”.
  • Error handling: Retry grab if failed, alert if medicine missing.
  • Flowchart logic: Command → Check database → Navigate → Detect & OCR → Arm pick → Deliver → Update logs → Return home.
  • Debugging: Test full sequence, edge cases, multiple objects, battery depletion during task.
  • Code size: Each phase of coding (init, navigation, vision, arm control, task flow) can easily accumulate 500+ lines of Python code.
