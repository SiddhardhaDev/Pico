Meet Pico

Pico is a state-of-the-art mobile robot designed to seamlessly integrate into your indoor environment. Equipped with a versatile robotic arm and omnidirectional wheels, Pico is engineered for precision, adaptability, and intelligent operation.

Key Features & Abilities

  • Advanced Robotic Arm: 6-axis robotic arm with high precision for intricate tasks, capable of lifting up to 2 kg.
  • AI-Powered Navigation: Cameras for accurate mapping, obstacle avoidance, and autonomous path planning.
  • Voice & Gesture Control: Intuitive control options for easy interaction and task assignment.
  • Modular Design: Expandable ports for additional sensors or tools, allowing for future upgrades and customization.
  • Long-lasting Battery: Up to 3 hours of continuous operation on a single charge, with quick-charge capabilities.
  • Object Recognition: Integrated computer vision for identifying and interacting with various objects.

Medicine Assistant Roadmap

Steps to enable Pico to recognize medicines and fetch them autonomously, with suggested tools and code hints.

Phase 1 – Core Movement & Arm Precision

  • Calibrate wheels for precise movement. Python + GPIO / motor library
  • Tune robotic arm to safely handle medicine boxes. Python + PWM / Servo control
  • Add limit switches or current sensors for safe grip. GPIO input / Python
  • Set a "home" docking position for start/end.

Phase 2 – Vision for Medicine Recognition

  • Use OCR to read medicine names. Tesseract or EasyOCR + Coral TPU
  • Create a medicine database with aliases for recognition.
  • Use object detection (YOLOv5 / MobileNet) to locate medicine boxes/bottles.
  • Add a LED ring for consistent lighting and better OCR accuracy.

Phase 3 – Autonomous Fetching Logic

  • Map key locations: medicine shelf, delivery point, table. SLAM libraries like RTAB-Map or OpenCV + ROS
  • Navigate safely with obstacle avoidance. Python + ROS or custom navigation code
  • Task flow: Command → Search → Navigate → Grab → Deliver.
  • Handle missing medicine: notify user via voice or dashboard.

Phase 4 – Safety, Reliability, and Refinement

  • Retry gripping if arm fails; alert if repeated failure.
  • Confirm OCR reads before grabbing the medicine.
  • Provide voice feedback: "Fetching..." and "Delivered". Python + pyttsx3 / gTTS
  • Optional: Face recognition to deliver medicine to a specific person. OpenCV + face_recognition library
